# sci_tmp 数据层

## 数据库分层概述

sci_tmp（Temporary Layer）是临时数据层，用于存储ETL处理过程中的中间计算结果和临时数据。作为数据加工处理的工作区域，TMP层承担着数据转换、清洗和聚合的重要职责。

**核心职能：**
- 存储ETL过程中的中间计算结果
- 支持数据清洗和转换的临时存储
- 为复杂计算提供分步结果保存
- 支持数据验证和质量检查

**设计原则：**
- 采用扁平化数据结构提升计算效率
- 支持海量数据的高并发写入
- 临时表使用后及时清理释放空间
- 可根据业务需求动态创建和销毁表结构

## 1. 数据表快速导航

sci_tmp层用于ETL过程中的临时数据存储，目前无预定义表结构。

在实际使用中，可根据业务需求动态创建临时表，常见类型包括：

| 临时表类型 | 说明 |
| ---------- | ---- |
| 中间结果表 | 存储复杂查询的分步计算结果 |
| 清洗临时表 | 用于数据质量规则校验和问题数据暂存 |
| 聚合临时表 | 支持多维度统计计算的中间状态存储 |
| 会话级临时表 | 仅在当前会话有效的临时计算结果 |
