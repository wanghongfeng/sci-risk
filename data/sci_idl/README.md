# sci_idl 数据层

## 数据库分层概述

sci_idl（Interface/Data Landing Layer）是数据仓库的接口层，用于存储从源系统获取的原始数据。作为数据进入数据仓库的第一站，IDL层承担着数据接入、原始存储和初步标准化的重要职责。

**核心职能：**
- 接收并存储来自外部系统的原始数据
- 保持数据的原始形态，不进行过多清洗转换
- 为后续数据处理提供基础数据源
- 支持数据溯源和原始数据恢复

**设计原则：**
- 保留数据原始字段和格式
- 添加数据接入时间戳便于追踪
- 支持增量数据和全量数据同步
- 与源系统数据结构保持一致或高度相似

## 1. 数据表快速导航

sci_idl层用于存储从源系统获取的原始数据，目前无预定义表结构。

在实际使用中，可根据业务需求创建数据表，常见类型包括：

| 数据表类型 | 说明 |
| ---------- | ---- |
