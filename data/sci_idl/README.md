# sci_idl 数据层

## 数据库分层概述

sci_idl（Interface/Data Landing Layer）是数据仓库的接口层，用于存储从源系统获取的原始数据。作为数据进入数据仓库的第一站，IDL层承担着数据接入、原始存储和初步标准化的重要职责。

**核心职能：**
- 接收并存储来自外部系统的原始数据
- 保持数据的原始形态，不进行过多清洗转换
- 为后续数据处理提供基础数据源
- 支持数据溯源和原始数据恢复

**设计原则：**
- 保留数据原始字段和格式
- 添加数据接入时间戳便于追踪
- 支持增量数据和全量数据同步
- 与源系统数据结构保持一致或高度相似

## 2. 字段详细说明

### risk_events（风险事件原始数据表）

存储从外部信源获取的风险事件原始数据，保持贴源存储。

| 字段名 | 类型 | 说明 |
| ------ | ---- | ---- |
| id | INT | 自增ID |
| source_channel | VARCHAR(50) | 来源渠道标签 |
| source_url | VARCHAR(500) | 原始来源URL |
| event_raw_text | TEXT | 原始文本 |
| event_raw_jsonb | JSONB | 原始数据载荷 |
| event_happen_timestamp | TIMESTAMP | 事件发生时间 |
| dw_create_timestamp | TIMESTAMP | 入库时间 |
