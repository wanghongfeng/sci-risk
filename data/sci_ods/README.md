# sci_ods 数据层

## 数据库分层概述

sci_ods（Operational Data Store）是操作数据存储层，用于存储经过清洗和标准化的运营数据。ODS层是数据仓库体系中的核心过渡层，承上启下，对原始数据进行质量提升和结构优化。

**核心职能：**
- 对IDL层原始数据进行清洗和标准化处理
- 实现跨源数据的一致性整合
- 建立基础数据质量规则和校验机制
- 为DW层提供高质量的业务数据

**设计原则：**
- 遵循3NF范式进行数据建模
- 统一数据编码和格式标准
- 保留数据变更历史追踪能力
- 确保与业务系统数据同步

## 1. 数据表快速导航

sci_ods层用于存储经过清洗和标准化的运营数据，目前无预定义表结构。

在实际使用中，可根据业务需求创建数据表，常见类型包括：

| 数据表类型 | 说明 |
| ---------- | ---- |
| 工厂主数据表 | 存储全球工厂信息 |
| 供应商主数据表 | 存储供应商层级和信用评级 |
| 产品主数据表 | 存储产品目录信息 |
| 物料清单表 | 存储产品BOM信息 |
| 供应路径表 | 存储供应链路径 |
| 关税规则表 | 存储进出口关税规则 |
| 风险映射表 | 综合评估供应链风险 |
| 销售订单表 | 存储销售订单信息 |
| 库存表 | 存储库存信息 |
| 订单表 | 存储客户订单信息 |
| 物流表 | 存储物流运输信息 |
